# PRD - Website Company Profile PT BPR Karawang Jabar (Perseroda)

## 1. Ringkasan Produk

Website **PT BPR Karawang Jabar (Perseroda)** adalah website company profile untuk memperkenalkan identitas, layanan, produk, pengurus, dan kanal kontak BPR secara profesional, kredibel, dan mudah diakses oleh masyarakat Karawang.

Website ini akan menjadi pusat informasi publik yang membantu calon nasabah memahami profil BPR, melihat produk/layanan yang tersedia, serta menghubungi pihak BPR melalui kanal resmi.

## 3. Tujuan Produk

- Menyediakan website company profile yang profesional untuk PT BPR Karawang Jabar (Perseroda).
- Meningkatkan kepercayaan masyarakat terhadap BPR sebagai lembaga keuangan daerah.
- Menampilkan informasi produk dan layanan secara jelas.
- Memudahkan calon nasabah menemukan alamat, jam layanan, telepon, email, dan kanal komunikasi.
- Menjadi fondasi awal untuk pengembangan fitur berikutnya seperti detail produk, form pengajuan, berita, atau admin panel.

## 4. Target Pengguna

### 4.1 Calon Nasabah

Masyarakat Karawang dan sekitarnya yang ingin mencari informasi produk tabungan, pinjaman, atau layanan BPR.

### 4.2 Nasabah Existing

Nasabah yang membutuhkan informasi kontak, jam operasional, lokasi kantor, atau informasi produk terbaru.

### 4.3 Mitra / Stakeholder

Pemerintah daerah, mitra bisnis, lembaga keuangan, dan pihak lain yang ingin mengetahui profil serta pengurus BPR.

### 4.4 Admin / Pengelola Website

Pihak internal atau developer yang akan mengelola konten, memperbarui informasi produk, dan menjaga website tetap berjalan.

## 5. Positioning Website

Website diposisikan sebagai website profil resmi/representatif yang menonjolkan:

- **Kepercayaan**: BPR sebagai lembaga keuangan yang profesional dan dapat dipercaya.
- **Kedekatan lokal**: BPR yang dekat dengan masyarakat Karawang, khususnya Cilamaya dan sekitarnya.
- **Kemudahan layanan**: informasi produk dan kontak dibuat mudah ditemukan.
- **Transparansi**: pengurus, alamat, jam layanan, dan produk ditampilkan jelas.

## 9. Kebutuhan Non-Fungsional

### 9.1 Performa

- Website ringan dan cepat dimuat.
- Asset CSS/JS tidak berlebihan.
- Gambar dikompresi sebelum production.

### 9.2 Keamanan

- File `.env` tidak boleh dapat diakses publik.
- Folder `application`, `system`, `vendor`, `storage`, dan `packages` tidak boleh diakses langsung dari browser.
- Form kontak wajib divalidasi.
- Input pengguna wajib dibersihkan/di-escape.
- Tidak menampilkan data sensitif di halaman publik.

### 9.3 Maintainability

- Struktur route, controller, dan view dibuat rapi.
- Layout utama dipisahkan dari halaman.
- Navbar/footer dibuat sebagai partial.
- Konfigurasi penting menggunakan `.env`.
- Dokumentasi disimpan di folder `docs/`.

### 9.4 Kompatibilitas

- Desktop modern browser
- Mobile browser
- Tablet
- PHP 8.1+
- Nginx + PHP-FPM

## 10. Struktur Teknis

## 14. Acceptance Criteria

Website versi awal dianggap selesai jika:

- Semua route utama dapat diakses dengan HTTP 200.
- Navbar dan footer tampil konsisten.
- Halaman Home menampilkan profil singkat, produk unggulan, pengurus, dan kontak cepat.
- Halaman Produk menampilkan TAHARA minimal sebagai produk awal.
- Halaman Kontak menampilkan alamat, telepon, email, dan jam buka.
- Website responsif di desktop dan mobile.
- URL bersih tanpa `index.php`.
- Folder sensitif tidak dapat diakses langsung.
- Tidak ada error PHP fatal di halaman publik.

## 15. Risiko dan Catatan

- Data referensi masih minim, sehingga beberapa konten awal perlu menggunakan copywriting sementara.
- Perlu konfirmasi apakah website ini akan menjadi website resmi atau hanya demo/redesign.
- Rakit v0.9.9 cukup lama, sehingga beberapa warning PHP 8.x perlu ditangani dengan hati-hati.
- Produk keuangan harus memakai bahasa yang akurat dan tidak menyesatkan.
- Klaim seperti “diawasi OJK”, “dijamin LPS”, atau legalitas lain hanya boleh ditampilkan jika ada data resmi.
